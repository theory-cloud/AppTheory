# M1 — Streamable HTTP runtime (Go) (Remote MCP / Claude)

Status: Complete  
Last updated: 2026-02-23  
Owner: AppTheory

This file is the work order + completion record for milestone **M1** of the “Claude-first Remote MCP” effort in AppTheory.

## Delivered (Go runtime)

- Streamable HTTP handler semantics on `runtime/mcp.Server.Handler()`:
  - `POST /mcp` accepts JSON-RPC **requests**, **notifications**, and **responses**
  - returns `202 Accepted` (no body) for accepted notifications/responses
  - `GET /mcp` supports resumability via `Last-Event-ID`
  - `DELETE /mcp` terminates a session (subsequent calls return `404`)
- JSON-RPC parsing:
  - `ParseRequest` accepts notifications (no `id`)
  - `ParseResponse` parses JSON-RPC responses
- SSE framing (Streamable HTTP):
  - server->client messages are `event: message` with `data: <JSON-RPC message>`
  - tool progress is sent as `notifications/progress` correlated by `_meta.progressToken`
  - SSE events include `id:` and can be replayed via `GET` + `Last-Event-ID`
- Stores + helpers:
  - `StreamStore` interface + `MemoryStreamStore` implementation (in-memory event log)
  - `OriginValidator` + `AllowOrigins(...)` helper

## Notes / deferred work

- No OAuth/DCR (Autheory) work in M1 (see M3).
- Keepalive strategy for long-idle streams is not yet standardized (see M2 guidance + infra constraints).
- `MemoryStreamStore` is intended for tests/local usage; durable event log storage is expected to be provided later (Dynamo/CDK in M2).
