# M2 — AWS streaming edge + CDK construct (Remote MCP / Claude)

Status: Complete  
Last updated: 2026-02-23  
Owner: AppTheory

This file is the work order + completion record for milestone **M2** of the “Claude-first Remote MCP” effort in AppTheory.

## Delivered (CDK)

- New CDK construct: `AppTheoryRemoteMcpServer` (jsii)
  - API Gateway **REST API v1**
  - Streamable HTTP mount at `/mcp`:
    - `POST /mcp` (response streaming enabled)
    - `GET /mcp` (response streaming enabled; `Last-Event-ID` replay)
    - `DELETE /mcp`
  - Optional DynamoDB tables:
    - session table (`sessionId` + TTL `expiresAt`; matches `runtime/mcp` Dynamo session store schema)
    - stream/event table (`sessionId` + `eventId` + TTL `expiresAt`; intended for durable resumable SSE)
  - Convenience env vars wired into the handler when possible:
    - `MCP_ENDPOINT`
    - `MCP_SESSION_TABLE`, `MCP_SESSION_TTL_MINUTES`
    - `MCP_STREAM_TABLE`, `MCP_STREAM_TTL_MINUTES`

## Documentation

- Added a Claude-first guide: `cdk/docs/mcp-server-remote-mcp.md`
- Updated CDK and repo API references to list the new construct.

## Notes / deferred work
