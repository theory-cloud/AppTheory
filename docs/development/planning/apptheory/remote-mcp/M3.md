# M3 — OAuth primitives + Autheory enablement (DCR day‑1)

Status: Complete  
Last updated: 2026-02-23  
Owner: AppTheory

This file is the work order + completion record for milestone **M3** of the “Claude-first Remote MCP” effort in AppTheory.

## Delivered (Go runtime)

New package: `runtime/oauth/`

Protected Resource (RFC9728 / MCP auth 2025-06-18):
- `ProtectedResourceWWWAuthenticate(resourceMetadataURL)` → builds `WWW-Authenticate: Bearer resource_metadata="..."`
- `ProtectedResourceMetadata` + `NewProtectedResourceMetadata(resource, authorizationServers)`
- `ResourceMetadataURLFromMcpEndpoint(mcpEndpoint)` helper (derives `/.well-known/oauth-protected-resource` from an `/mcp` URL)
- `RequireBearerTokenMiddleware(...)` (AppTheory middleware) for `Authorization: Bearer ...` enforcement + challenge response
- AppTheory handlers for metadata docs:
  - `ProtectedResourceMetadataHandler(...)`
  - `AuthorizationServerMetadataHandler(...)`

Authorization Server (Autheory enablement):
- `AuthorizationServerMetadata` + `NewAuthorizationServerMetadata(issuer)` (RFC8414)
- RFC7591 DCR types + policy helper + validation:
  - `DynamicClientRegistrationRequest`
  - `ValidateDynamicClientRegistrationRequest(...)`
  - `ClaudeDynamicClientRegistrationPolicy()`
- PKCE utilities (RFC7636):
  - `NewPKCECodeVerifier()`
  - `PKCEChallengeS256(...)` / `PKCEVerifyS256(...)`
- Token/code primitives intended for AS implementations and tests:
  - `NewOpaqueToken()`
  - `AuthorizationCodeStore` + `MemoryAuthorizationCodeStore`
  - `RefreshTokenStore` + `MemoryRefreshTokenStore`

## Delivered (Testkit)

New package: `testkit/oauth/`

- `ClaudePublicClient` harness that emulates Claude connector OAuth behavior:
  - parses `WWW-Authenticate` → fetches protected resource metadata → fetches AS metadata
  - performs DCR → Authorization Code + PKCE → token exchange → refresh
- Includes an in-repo reference flow test using `httptest` servers.

## Delivered (CDK)

New construct: `AppTheoryMcpProtectedResource`

- Adds `GET /.well-known/oauth-protected-resource` (RFC9728) to an `AppTheoryRestApiRouter`
- Uses an API Gateway mock integration (no additional Lambda) and supports CDK tokens via `Stack.toJsonString(...)`

## Notes / deferred work

- This milestone does **not** implement a full OAuth Authorization Server for Autheory; it provides the primitives and harness needed to do so.
- Authorization server CDK building blocks (tables, endpoint wiring, etc.) are expected in later milestones.
